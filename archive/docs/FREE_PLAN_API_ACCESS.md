# X (Twitter) Developer Portal 無料プランでのAPIアクセス

## 無料プランでもAPIは使用可能

**重要**: Twitter Developer PortalのBasicプラン（無料）でも、APIは使用できます。

## 無料プランでの制限

### 使用可能な機能

- ✅ **基本的な投稿機能**: `create_tweet`（ツイート投稿）
- ✅ **アカウント情報取得**: `get_me`（自分のアカウント情報）
- ✅ **ツイート取得**: `get_users_tweets`（ユーザーのツイート取得）
- ✅ **API v2の基本的なエンドポイント**

### 制限事項

- ⚠️ **レート制限**: 無料プランではレート制限が厳しい
  - ツイート投稿: 1時間あたり300ツイート（通常は十分）
  - アカウント情報取得: 1時間あたり75回
- ⚠️ **一部のエンドポイントが制限**: 高度な機能は有料プランが必要
- ⚠️ **API v2の一部機能**: 一部の高度な機能は制限される可能性

## 403エラーの原因（無料プランとは無関係）

403エラーは、無料プランによる制限ではなく、通常は以下の原因です：

### 1. アプリの権限設定

- **原因**: アプリの権限が「Read」のみに設定されている
- **解決方法**: 「Read and write」に変更し、Access Tokenを再生成

### 2. Access Tokenの再生成不足

- **原因**: 権限変更後にAccess Tokenを再生成していない
- **解決方法**: Access TokenとAccess Token Secretを再生成

### 3. X側の承認待ち

- **原因**: 新規作成や権限変更後、X側の承認が必要
- **解決方法**: 承認を待つ（通常、数時間〜数日）

### 4. アプリの状態

- **原因**: アプリが停止（SUSPENDED）されている
- **解決方法**: アプリを再アクティブ化、または新しいアプリを作成

## 無料プランで使用可能なエンドポイント

このプロジェクトで使用しているエンドポイントは、無料プランでも使用可能です：

- `GET /2/users/me` - アカウント情報取得 ✅
- `POST /2/tweets` - ツイート投稿 ✅
- `GET /2/users/:id/tweets` - ツイート取得 ✅

## 確認方法

### アプリの状態を確認

1. Twitter Developer Portalでアプリを開く
2. 「Overview」タブでアプリの状態を確認
3. 以下を確認：
   - アプリの状態が「Active」か
   - 「Pending approval」などの表示がないか
   - エラーメッセージや警告が表示されていないか

### 権限設定を確認

1. 「Settings」タブを開く
2. 「App permissions」セクションを確認
3. 「Read and write」が選択されているか確認

## 結論

**無料プランでもAPIは使用可能です。** 403エラーは、無料プランによる制限ではなく、アプリの権限設定やX側の承認の問題である可能性が高いです。

## 次のステップ

1. アプリの状態を確認（「Active」「Pending approval」「SUSPENDED」など）
2. 権限設定を確認（「Read and write」に設定されているか）
3. Access Tokenを再生成（権限変更後）
4. 詳細なエラーメッセージを確認（GitHub Actionsのログ）
