# テスト投稿の実行方法

## 投稿の削除について

**はい、可能です！** GitHub Actionsから投稿したツイートは、スマホのXアプリから削除できます。

- X（Twitter）の投稿は、どのデバイスからでも削除可能です
- スマホのXアプリで投稿を開いて「削除」を選択するだけです

## テスト投稿の実行方法

### 方法1: GitHub Actionsでテスト投稿（推奨）

1. **GitHubリポジトリにアクセス**
   - https://github.com/kazzyass17-commits/blog-to-twitter-bot

2. **「Actions」タブを開く**
   - リポジトリページの上部タブから「Actions」をクリック

3. **「Test Post to Twitter」ワークフローを選択**
   - 左サイドバーから「Test Post to Twitter」をクリック

4. **「Run workflow」ボタンをクリック**
   - 右上の「Run workflow」ボタンをクリック
   - 「Run workflow」をクリックして実行

5. **実行結果を確認**
   - ワークフローが実行されます
   - データベースが初期化され、テスト投稿が実行されます

6. **投稿を確認**
   - Xアプリで投稿を確認
   - 問題があれば、スマホから削除できます

### 方法2: ローカル環境でテスト投稿

```powershell
cd c:\Users\kazz17\.cursor\blog-to-twitter-bot-temp

# データベースを初期化（初回のみ）
python init_posts_from_index.py

# テスト投稿（実際に投稿）
python test_post.py --account both
```

## テスト投稿の内容

- **365botGary**: ブログからランダムに1件選択して投稿
- **pursahsgospel**: ブログからランダムに1件選択して投稿

## 投稿後の確認

1. **Xアプリで投稿を確認**
   - @365botGary と @pursahsgospel のアカウントを確認
   - 投稿が正しく表示されているか確認

2. **問題がある場合**
   - スマホのXアプリで投稿を開く
   - 「削除」を選択して削除

## 注意事項

- **テスト投稿は実際に投稿されます**
- 投稿後、スマホから削除できます
- データベースに記録されるため、次回の投稿では同じ内容は選ばれません（1巡するまで）

## 次のステップ

テスト投稿が成功したら：

1. **スケジュール実行を開始**
   - GitHub Actions: 自動的に8時、14時、20時 JSTに実行されます
   - ローカルPC: `python schedule.py` を実行

2. **定期的な確認**
   - 投稿が正しく実行されているか確認
   - 問題があれば、スマホから削除
